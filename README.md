# StarryAI

StarryAI 是一个面向本地开发与实验的节点式 AI 工作流工作台，提供可视化编排、运行控制与运行态观察能力。

## 项目简介

StarryAI 采用前后端分离架构：后端负责图校验、运行调度、事件与指标输出，前端提供可视化工作台，用于编辑图、配置节点、保存图并发起运行。

它适合用来搭建和验证 AI 工作流原型，尤其适合需要以下能力的场景：

- 用画布方式组织节点和连线，而不是手写流程配置
- 在运行前检查图结构与节点配置是否合法
- 在运行中查看状态、事件、指标和诊断信息
- 通过 Secret 管理敏感配置，避免在图中直接保存明文
- 保存和复用本地工作流图

## 核心能力

- 可视化图编辑：基于画布创建节点、连接节点、调整布局，并维护图的基本结构
- 节点配置面板：通过表单或结构化配置编辑节点参数
- 图校验：在保存或运行前检查图结构、端口连接和配置合法性
- 运行控制：直接从工作台启动、停止并跟踪一次图运行
- 运行态观察：查看事件流、运行指标和诊断信息，辅助排查问题
- Secret 管理：集中维护敏感配置，并在节点配置中以引用方式使用
- 图持久化：保存、加载、删除本地工作流图，便于重复使用和迭代

## 技术架构

StarryAI 由两个主要部分组成：

- Backend：Python 3.12 + FastAPI，负责图校验、运行调度、REST/WS 接口和运行数据输出
- Frontend：React + TypeScript + Vite，结合 React Flow 和 Zustand 提供工作台交互体验

整体运行方式是本地启动后端服务和前端工作台，通过 HTTP 和 WebSocket 完成图配置、运行控制与运行态信息同步。

## 适用场景

- AI 工作流原型设计与验证
- 节点式编排工具的本地实验环境
- 需要可视化编辑和运行观察能力的内部工具雏形
- 需要把敏感配置与流程图本身分离管理的本地项目

## 快速开始

### 环境要求

- Python `3.12`
- Node.js 与 `npm`

### 推荐方式：一键启动

在仓库根目录执行：

```bash
python main.py
```

启动后会同时拉起后端和前端开发环境。默认情况下：

- 后端地址：`http://127.0.0.1:8000`
- 前端地址：`http://127.0.0.1:5173`

如果默认端口被占用，启动器会自动切换到可用端口，并在终端中打印实际访问地址。

启动器还会在缺少基础依赖时尝试补齐：

- Python 运行依赖：通过 `requirements.txt`
- 前端依赖：通过 `frontend/package.json`

可查看帮助信息：

```bash
python main.py --help
```

## 手动启动方式

如果你希望分别控制前后端进程，可以按下面的方式启动。

### 1. 安装依赖

在仓库根目录安装 Python 依赖：

```bash
python3.12 -m pip install -r requirements.txt
```

安装前端依赖：

```bash
cd frontend
npm install
```

### 2. 启动后端

```bash
cd backend
python3.12 -m uvicorn app.main:app --reload
```

后端默认地址为：

```text
http://127.0.0.1:8000
```

### 3. 启动前端

在新的终端中执行：

```bash
cd frontend
npm run dev
```

前端默认地址为：

```text
http://127.0.0.1:5173
```
